# absent-thesis
: arXiv Paper Topic Dynamics Tracking

## 1. 프로젝트 개요 및 진척도 (Progress & Overview)

본 프로젝트는 arXiv 데이터셋을 활용하여 시간에 따른 학술 연구 주제의 변화를 체계적으로 분석하는 시스템 구축을 목표로 합니다. 트랜스포머 모델을 기반으로 한 자연어 처리 기술을 적용하여 연구 동향의 패턴을 발견하고 시각화하는 것을 주요 내용으로 합니다.

| 구분 | 진척도 | 주요 성과 |
| :--- | :--- | :--- |
| **1주차 목표** | **100% 완료** | 데이터 샘플링, 전처리, 임베딩 생성 및 탐색적 데이터 분석(EDA) 완료. |
| **핵심 성과** | **1년 단위 타임 슬롯** 전략 확정 및 적용. |
| **다음 목표** | UMAP/HDBSCAN 파라미터 **AutoML 최적화** 및 시기별 클러스터링 실행. |

## 2. 방법론 및 핵심 전략 (Methodology & Core Strategy)

### 2.1 데이터 전처리 및 타임 슬롯 전략

| 항목 | 세부 내용 | 전략적 의의 |
| :--- | :--- | :--- |
| **임베딩 모델** | **Sentence-BERT (all-mpnet-base-v2)** 사용 | 논문의 **제목과 초록의 의미론적 유사도**를 고차원 벡터로 변환하여 기존 키워드 기반 분석의 한계를 극복. |
| **데이터 샘플링** | **연도별 균등 샘플링 (1,000건/년)** | 클러스터링 결과의 안정성 확보 및 시기별 비교의 공정성 유지. |
| **타임 슬롯** | **1996년 이후 1년 단위 분할** | 데이터 안정성이 확인된 구간을 **1년** 단위로 분할하여 **최대 시간 해상도** 확보 (예: `1996-1996`, `2020-2020`). |

### 2.2 분석 파이프라인 (향후 진행 방식)

1.  **데이터 분할**: 전체 데이터를 30여 개의 `time_period` (1년 단위)로 분할.
2.  **UMAP**: 각 타임 슬롯별 데이터에 대해 **개별적으로** 적용하여 차원 축소.
3.  **HDBSCAN & AutoML**: 각 타임 슬롯별로 HDBSCAN을 적용하며, **Optuna**를 활용하여 `min_cluster_size` 등의 하이퍼파라미터를 **독립적으로 튜닝**하여 최적의 클러스터 품질(실루엣 계수 등)을 확보.
4.  **트렌드 추적**: 시기별 클러스터의 **중심 벡터(Centroid)**와 **규모(Volume)**를 추출하여 토픽의 생성, 성장, 의미 변화를 추적.

## 4. 교수님 질문에 대한 답변

### 4.1. 연도별 트렌드 추출 방식 (Q1)

구글의 Logistic Regression 방식 대신 의미 기반 클러스터링 및 진화 추적 방식을 사용합니다.

1.  **트렌드 정의**: 트렌드를 **'특정 시기에 집중된 연구 주제의 규모 변화'** (Volume Trend)와 **'주제 자체의 의미론적 위치 변화'** (Semantic Drift) 두 가지로 정의합니다.
2.  **Volume Trend 추출**: 매년(1년 슬롯) HDBSCAN으로 군집화된 **각 토픽 클러스터의 논문 수**를 시계열로 추적합니다. (선 그래프로 시각화).
3.  **Semantic Drift 추출**: 매년 추출된 각 토픽 클러스터의 **중심 벡터(Centroid)**를 추출한 후, **년도 $t$의 중심 벡터**와 **년도 $t+1$의 중심 벡터** 간의 **코사인 유사도 또는 유클리드 거리**를 계산하여 토픽 의미의 변화 속도를 정량화합니다.

### 4.2. HDBSCAN 사용 특별한 이유 (Q2)

차원이 UMAP으로 축소되었음에도 HDBSCAN을 사용하는 이유는 **'대용량 처리'** 목적보다는 **'토픽 모델링에 적합한 품질'** 확보 때문입니다.

1.  **비구형 클러스터 적합성**: UMAP으로 축소된 임베딩 공간은 복잡하고 **비구형(Arbitrary Shape)**의 클러스터 형태를 가집니다. HDBSCAN은 밀도 기반으로 이러한 형태를 가정 없이 정확하게 찾아냅니다.
2.  **노이즈(Outlier) 강건성**: 학술 논문에는 **신규 출현 토픽**이나 **매우 틈새 연구** 등 단일 논문 클러스터가 많습니다. HDBSCAN은 이들을 **명시적으로 노이즈(Noise)**로 분류하여 핵심 트렌드 클러스터를 왜곡하는 것을 방지합니다.

### 4.3. 토픽 라벨링 방식 (Q3)

**c-TF-IDF (Class-based TF-IDF)** 방식을 사용합니다.

1.  **원리**: HDBSCAN으로 군집화된 **각 클러스터**를 하나의 거대한 **'문서(Class)'**로 간주합니다.
2.  **특징 추출**: 이 '문서(클러스터)' 내에서 단어 빈도(TF)를 계산하고, 전체 클러스터(문서군) 간의 역문서 빈도(IDF)를 계산하여 각 클러스터를 **가장 잘 대표하는 상위 키워드 N개**를 추출합니다.
3.  **장점**: 이 방식은 Sentence-BERT의 의미 기반 클러스터링 결과(토픽)에 **가장 식별력이 높고 인간이 해석 가능한 라벨**을 붙여주어, 트랜스포머 기반의 의미 분석을 완성합니다.

### 4.4. 의미 공간의 변화 포착 가능성 (Q4)

1.  **의미 $=$ 벡터 위치**: 단어(토픽)의 의미를 **임베딩 공간 상의 벡터 위치**로 정의합니다. "gay"라는 단어의 의미가 변한다는 것은, 해당 단어와 관련된 논문들이 군집화되는 **클러스터의 중심 벡터($C$)**가 이동한다는 것을 의미합니다.
2.  **변화 포착 메커니즘**:
    * **시기별 클러스터 매칭**: 연도 $t$의 클러스터 $C_t$와 연도 $t+1$의 클러스터 $C_{t+1}$ 사이의 **최소 코사인 거리**를 기준으로 **토픽 연속성(진화 경로)**을 설정합니다.
    * **의미 변화 정량화**: 진화 경로가 연결된 $C_t$와 $C_{t+1}$의 코사인 유사도를 계산하고, 이를 **1에서 뺀 값 (Cosine Dissimilarity)**을 **'토픽 의미 변화율(Semantic Drift Rate)'**로 정의하여 시계열로 추적합니다.

이 방식을 통해 단순한 트렌드 변화(논문 수 증가)를 넘어, **"특정 용어(토픽)를 다루는 연구의 내용적/맥락적 변화"**를 정량화하여 설명할 수 있습니다.
